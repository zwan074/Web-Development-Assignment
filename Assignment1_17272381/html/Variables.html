Javascript Variables

Variable

A variable is a “named storage” for data. We can use variables to store objects.

To create a variable in JavaScript, we need to use the let keyword.
Example below shows a variable with the name “message”:

let message;

Now some data could be add into it by using the assignment operator =:

let message;

message = 'Hello'; // store the string

The string is now saved into the computer memory associated with the variable. Then it can be accessed by using the variable name:

let message;
message = 'Hello';

alert(message); // shows the variable content

We can also merge the variable declaration and assignment into a single line:

let message = 'Hello'; // define assign the value to the variable

alert(message); // Hello!

Constants

use const instead of let to declare a constant (unchanging) variable.

const myBirthday = '19.06.1984';

Variables declared using const are called “constants”. They are immutable. An attempt to do it would cause an error:

const myBirthday = '16.06.1984';

myBirthday = '18.04.2000'; // error, can't reassign the constant!

We can use syntax const to guarantee it, and advise other people it can not be changed.

Name things right

One more extremely important thing is to assign good name for variables.

Take time to think if needed and name the variables sensibly.

Variable naming is one of the most important and complex skills in programming. A quick glance at variable names can reveal which code is written by a beginner and which by an experienced developer.

In a real project, most of the time is spent on filing , modifying and extending the existing code base, it’s much easier to help programmers understanding the code with right name of variables.

Spending time thinking about right name of variables will help recieve greate return in long term

Some good practices are:

  • Use human-readable names like userName or passWord.
  • Stay away from abbreviations or short names like a, b, c.
  • Make the name descriptive and concise when people reading at first instances. Examples of bad names are data and value. Such a name is meaningless.
  • Agree on terms within your team. If a site visitor is called a “user” then we should name related variables like currentUser or newUser , but not currentVisitor or a newManInTown .

Summary

We can declare variables to store data. That can be done using let or const.

  • let – is a modern variable declaration.
  • const – is like let, however the value of the variable can’t be changed.

Variables should be named in a way that allows us to easily understand its semantics.
